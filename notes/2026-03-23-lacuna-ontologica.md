## 1. A lacuna ontológica no Foundations.md

O documento trata extensivamente da epistemologia (dados-como-construídos, Sandelowski) e desce para metodologia/técnicas (quantitização, qualitização), mas **nunca explicita a camada ontológica** que precede tudo.

A hierarquia filosófica (Crotty, 1998, via Herrera Batista Cap 7):

```
ontologia → epistemologia → perspectiva teórica → metodologia → métodos
```

Pergunta ontológica ausente no Foundations.md:
> Quando dizemos "dado qualitativo", estamos dizendo que existe uma entidade no mundo com essa natureza? Ou estamos dizendo que um pesquisador escolheu representar algo dessa forma?

## 2. Cada método carrega uma ontologia embutida

Insight do Prof. Altay de Souza: cada método não é neutro — carrega um pacote ontológico-epistemológico completo.

- **Quantitativo** → ontologia **realista**: existe uma realidade objetiva lá fora, dados são entidades que existem independentes do observador, a gente "coleta"
- **Qualitativo** → ontologia **construtivista/relativista**: realidade é múltipla, construída socialmente, cada indivíduo co-constrói significado — dados são "tomados", não "dados"

Quando você escolhe um método, está comprando o pacote ontológico junto, queira ou não.

## 3. Transformação de dados como travessia ontológica

Sacada central: quando se faz **crossover mixed analysis** (análise quanti de dados quali, ou vice-versa), não é apenas uma operação técnica. É uma **travessia ontológica** — uma passagem entre visões de mundo diferentes.

Quantitizar um dado qualitativo significa forçá-lo a se comportar como se existisse numa realidade objetiva (ontologia realista), quando ele foi construído numa ontologia construtivista. E vice-versa na qualitização.

Essa tensão ontológica é o que ninguém está discutindo explicitamente na literatura de mixed analysis. Seria contribuição original.

## 4. Paralelo com Herrera Batista (2021)

| Herrera Batista (design) | Mixed Analysis |
|---|---|
| O que é o ato de **projetar**? | O que é o ato de **analisar**? |
| Componentes: objeto, designer, usuário | Componentes: dado, pesquisador, método |
| Cada abordagem de design carrega uma worldview | Cada método analítico carrega uma ontologia |
| Designer transforma necessidade em objeto | Pesquisador transforma experiência em dado |

Ambos são atos de construção que carregam pressupostos sobre o que é real.

O Cap 7.3 do Herrera Batista mapeia cada paradigma (positivista, pós-positivista, interpretativo, construtivista) nas três dimensões (ontológica, epistemológica, metodológica). Esse mapeamento é exatamente o que falta no Foundations.md — o Sandelowski fala de worldviews (Seção 1.2) mas nunca destrincha a camada ontológica.

## 5. Data governance como ponte com o mundo corporativo

A metáfora "dados são o novo petróleo" (Clive Humby, 2006) carrega a epistemologia de dados-como-dados — trata dados como recurso natural que existe no mundo, esperando ser extraído e refinado. É a visão positivista em forma de slogan.

Tensão produtiva:
- **Data governance** assume que dados existem e precisam ser governados (qualidade, linhagem, acesso) — perspectiva operacional
- **Epistemologia de métodos mistos** pergunta: quem decidiu que isso é um dado? Que categorias usar? O que ficou de fora? — perspectiva reflexiva

Não são contraditórios, são complementares em níveis diferentes. Data governance governa o artefato; epistemologia questiona o processo que criou o artefato.

Conexão com Seção 4.2 do Foundations.md: a contagem como "tecnologia disciplinar" (Martin, 2004) que torna objetos "dóceis". Data governance é a institucionalização dessa disciplina.

## 6. Posicionamento do tema central

O trabalho não é sobre **mixed methods** no sentido convencional (design-level). É sobre **mixed analysis** — integração de abordagens analíticas no nível da análise.

Cadeia conceitual:
```
mixed analysis → transformação de dados → quantitização + qualitização
```

Esse é um tema recente e de nicho (Onwuegbuzie & Johnson, 2021; Onwuegbuzie, 2025), pouco discutido mesmo na literatura especializada, e praticamente ausente em ferramentas.
